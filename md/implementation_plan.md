# 籌碼面分析建議實作計畫

這個計畫旨在根據目前獲取的籌碼面數據（融資餘額與三大法人），提供使用者具體的「持倉 / 減倉 / 空倉」建議與相關動態說明。

## Proposed Changes

### [MODIFY] `index.html`

我們將修改 `renderChipsSection` 函數（約 4486 行開始），在渲染完原本的「融資餘額」與「三大法人」數據卡片後，根據這些數據計算出一個「籌碼面綜合評分」，並在區塊底部顯示具體的判斷說明與操作建議。

**評分判斷邏輯：**
1. **外資**：連續買超 $\ge$ 3天 `+1` 分；連續賣超 $\le$ 3天 `-1` 分。
2. **投信**：連續買超 $\ge$ 2天 `+1` 分；連續賣超 $\le$ 2天 `-1` 分。
3. **融資**：趨勢為持續減少（籌碼安定） `+1` 分；趨勢為連續增加（散戶浮額增） `-1` 分。

**給出建議標準：**
*   **分數 $\ge$ 1**：籌碼面偏多。建議：**持倉 (Hold)** （法人資金進駐或籌碼安定）
*   **分數 $==$ 0**：籌碼面中性。建議：**觀望 (Watch)** （多空力道抵銷或無明顯表態）
*   **分數 $==$ -1**：籌碼面偏空。建議：**減倉 (Reduce)** （法人有倒貨跡象或散戶籌碼增加）
*   **分數 $\le$ -2**：籌碼面極空。建議：**空倉 (Empty)** （法人連續倒貨且散戶接盤，風險極高）

**UI 修改部分：**
1. 將原本寫死在 HTML 中 `loadDecisionPanel` 生成的提示 `💡 外資連買 3 天以上為正面訊號；融資上升但股價不漲則需注意風險。` 取代為**動態生成**的專屬建議區塊。
2. 在 `renderChipsSection` 結尾處插入一個 `alert` 樣式的區塊，列出：
   - 綜合判斷理由（例如：「外資連續買超」、「融資持續減少(籌碼安定)」等）。
   - 具體操作建議（持倉/減倉/空倉）。
