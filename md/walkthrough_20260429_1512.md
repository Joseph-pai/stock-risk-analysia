# 實作完成說明 (Walkthrough)

## 任務摘要
本次任務解決了上櫃公司（如 8455 大拓-KY）名稱識別缺失與籌碼資料歸零的問題，同時優化了搜尋 UI，新增「搜尋範圍鎖定」機制。

## 修改內容詳述

### 1. 後端公司類型檢測 (`check-company-type.js`)
- **更新 API**：將上櫃公司資料來源改為 `mopsfin_t187ap03_O`，此 API 包含全量上櫃股票（含 KY 股），解決了 8455 找不到名稱的問題。
- **新增範圍鎖定**：支援 `scope` 參數，當前端鎖定特定範圍（如上櫃）時，後端會優先/僅在該範圍內檢測。

### 2. 後端籌碼資料抓取 (`fetch-chips.js`)
- **日期轉換**：新增 `getTaiwanDate()` 函數，可將西元日期自動轉換為民國年格式（如 `115/04/28`），以符合櫃買中心 API 格式。
- **路由分配**：根據 `company_type` 參數，當識別為 `otc` 時，備援邏輯會自動導向櫃買中心的 `margin_bal_result.php` 與 `3itrade_hedge_result.php`，確保上櫃股票不再出現籌碼資料為 0 的情況。

### 3. 前端界面優化 (`index.html`)
- **搜尋範圍鎖定**：
  - 點擊「上市/上櫃/興櫃」按鈕時，會立即觸發搜尋建議的過濾。
  - 當鎖定為「上櫃」時，搜尋建議僅顯示上櫃相關內容（若有清單），且檢測時會強制鎖定範圍。
- **參數傳遞**：所有籌碼資料請求現在都會帶上 `company_type`，確保後端能正確分流。

- **籌碼資料**：經由測試 URL 確認能正確抓取到三大法人買賣超與融資餘額數據。
